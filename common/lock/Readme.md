# fs4me-lock

## MultiLock

Многопоточная блокировка на основе атомарной операции `FailIfExist` с поддержкой драйверов, реализующих trait `Driver`.


### Типы блокировок

| Режим | Описание | Для чего подходит |
|-------|----------|------------------|
| **Read** | Множественный доступ — много процессов одновременно | Чтение данных без блокировки записей |
| **Write** | Эксклюзивный доступ — только один процесс | Взаимоисключающий доступ к данным |
| **WriteQueue** | Очередь — процессы встают в очередь | Распределённые системы с низкой загрузкой |

### Принцип работы

1. Получает уникальный доступ к файлу блокировки
2. При успехе пробует добавить запись с блокировкой, если режим блокировки позволяет добавить запись, учитывая другие блокировки
3. Время жизни блокировки: 5 минут после последнего обновления (периодически обновляется в фоне)
4. При выходе из области видимости блокировка автоматически снимается

### Формат lock-файла

```text
1234_1=1780118532000000000=write_queue
1234_2=1780118532000000000=write_queue
1234_3=1780118532000000000=read
1234_4=1780118532000000000=read
```


### Примеры

#### Блокировка чтения (множественный доступ)

```rust,ignore
let lock = MultiLock::try_lock(
    uuid,
    driver,
    "/path/to/file.lock",
    LockMode::Read,
)?;

// Читаем данные ...
// При выходе блокировка снимается
```

#### Блокировка записи (эксклюзивный доступ)

```rust,ignore
let lock = MultiLock::try_lock(
    uuid,
    driver,
    "/path/to/file.lock",
    LockMode::Write,
)?;

// Записываем данные ...
// При выходе блокировка снимается
```

#### Блокировка записи в очередь

```rust,ignore
let lock = MultiLock::try_lock(
    uuid,
    driver,
    "/path/to/file.lock",
    LockMode::WriteQueue,
)?;

// Ждём блокировки ...
// При выходе блокировка снимается
```

## BaseLock

Атомарная блокировка на основе `FailIfExist` операции (создать файл только если его не существует). Работает с поддержкой драйверов, реализующих trait `Driver`.

### Принцип работы

1. Попытка создать lock-файл через атомарную `FailIfExist` операцию
2. Если файл уже существует:
   - Если UUID в файле принадлежит текущему клиенту → успешно
   - Иначе блокировка занята
3. Если срок жизни блокировки истёк:
   - Удаляется файл
   - Повторно пробуем установить блокировку

### Время жизни
Блокировка обновляется в фоновом режиме.
Если блокировка не обновлялась более 5 минут, то считается недействительной.

### Формат lock-файла

```text
<FsUuid>
```

Пример: `12345678_98765`

UUID:
- `<connection_id>` — идентификатор клиента
- `<copy_id>` — идентификатор копии клиента

### Drop

При выходе из области видимости:
- Останавливается фоновая задача обновления
- Удаляется lock-файл
